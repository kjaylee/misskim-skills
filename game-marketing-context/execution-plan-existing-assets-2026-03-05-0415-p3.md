# 기존 자산 마케팅 실행안 (P3, 7일)

- 작성 시각: 2026-03-05 04:15 KST
- 범위: 신규 개발 없이 기존 게임 자산(딥링크/영상/스크린샷/채널)만 재배치
- 대상 자산: 텔레그램 Mini App 포트폴리오 상위 3개 게임(최근 14일 DAU 기준)

## 1) Objective (목표)

**7일 내 기존 자산 재활용으로 신규 플레이어 유입과 첫 플레이 전환율을 동시 개선한다.**

- 1차 목표: 신규 플레이어 수 +30% (직전 7일 대비)
- 2차 목표: 딥링크 클릭→게임 시작률 55% 이상
- 3차 목표: Rented 채널 유입의 Owned 채널 전환률 12% 이상

## 2) Channel Mix (채널 믹스)

### Owned (55%)
1. 텔레그램 포트폴리오 채널
   - 고정 메시지: "이번 주 추천 1게임" 단일 CTA
   - 주 4회 포스트(성과 카드/챌린지/보상 공지/리더보드)
2. 게임 내 크로스프로모션 슬롯
   - 게임오버/메인메뉴 배너 1개씩 교체(기존 배너 소재 재사용)

### Rented (35%)
1. TikTok + YouTube Shorts
   - 기존 영상 재편집 5개(15~25초)
   - 훅 A/B(도전형 vs 결과형) 테스트
2. Reddit (r/IndieGaming, r/WebGames)
   - 주 2회 포스트, 1회는 플레이 영상/GIF 중심, 1회는 개발 스토리 중심

### Borrowed (10%)
1. 텔레그램 소규모 커뮤니티 3곳 교차 소개
2. 마이크로 크리에이터 2명에게 기존 게임 코드/리워드 제공 후 플레이 유도

## 3) 7-Day Action Schedule

| Day | 핵심 액션 | 산출물 |
|---|---|---|
| D1 (세팅) | 상위 3개 게임 선정, UTM 규칙 확정, 기존 크리에이티브 10개 선별 | 추적 시트 1개, 소재 리스트 1개 |
| D2 | 텔레그램 포스트 #1(도전형 CTA), Shorts 1개 게시 | 게시물 2개, 초기 CTR 데이터 |
| D3 | Shorts 1개 + Reddit 포스트 #1, 게임 내 배너 교체 | 외부 유입 2개 소스 추가 |
| D4 | 중간 점검(하위 30% 소재 컷), 텔레그램 포스트 #2(보상형 CTA) | 컷/유지 결정표 |
| D5 | Shorts 2개 연속 업로드, 커뮤니티 2곳 교차 소개 | 유입 피크 실험 |
| D6 | Reddit 포스트 #2 + 크리에이터 2명 아웃리치 완료 | Borrowed 채널 유입 확보 |
| D7 (마감) | KPI 집계, 승리 조합 확정, 다음 7일 확장안 정의 | 주간 리포트 + 확장 백로그 |

## 4) KPI Targets

### 퍼널 KPI
- 텔레그램 포스트 CTR: **≥ 5.0%**
- Shorts/TikTok 링크 CTR: **≥ 2.0%**
- 딥링크 클릭→게임 시작률: **≥ 55%**
- 신규 플레이어 수: **+30%** (직전 7일 대비)

### 품질 KPI
- D1 리텐션(신규 코호트): **≥ 23%**
- 공유 버튼 클릭률: **≥ 8%**
- Rented→Owned 전환률(채널 구독/그룹 유입): **≥ 12%**

### 운영 KPI
- 소재 제작 리드타임: **24시간 이내**
- 저성과 소재 교체 시간: **12시간 이내**

## 5) Risk Controls (리스크 통제)

1. **채널 편중 리스크**
   - 통제: 하루 단위로 Owned/Rented 유입 비중 점검, 단일 채널 비중 60% 초과 시 다음날 집행 제한

2. **스팸/커뮤니티 제재 리스크 (Reddit/텔레그램 그룹)**
   - 통제: 홍보 글 비중 50% 이하 유지, 질문/피드백형 문장 포함, 서브레딧 규칙 체크리스트 사전 통과 후 게시

3. **광고 피로/유저 반감 리스크**
   - 통제: 동일 CTA 연속 2회 이상 금지, 카피 3종 로테이션, 과도한 보상 미끼 표현 금지

4. **트래킹 누락 리스크**
   - 통제: 게시 전 UTM 프리플라이트 체크(소스/캠페인/크리에이티브), 누락 건은 집계에서 분리 표기

5. **전환 저하 리스크(클릭은 높고 시작률 저조)**
   - 통제: 단일 링크 정책 유지, 랜딩 카피와 게임 첫 화면 메시지 일치화, D4/D7에 시작률 원인 점검

## 6) 운영 원칙

- 신규 기능 개발/디자인 리소스 추가 투입 없이 **기존 자산 재활용 우선**
- 의사결정은 감이 아닌 **일일 KPI 스냅샷** 기준
- D7 종료 시 승리 조합 1개만 다음 주 확장(바벨 전략: 안정 80 / 실험 20)
