# 기존 자산 마케팅 실행안 (P3)

- 작성일: 2026-03-04 (KST)
- 목적: **신규 게임/툴 제작 없이**, 현재 보유한 텔레그램 Mini App 게임 포트폴리오의 48시간 유입 증대
- 대상 자산(기존):
  1. 이미 운영 중인 게임 딥링크
  2. 기존 플레이 영상/스크린샷
  3. 기존 채널(텔레그램/블로그)

## 1) 유입 채널 2개

### 채널 A — Telegram Owned Channel (직접 보유 채널)
- 자산 활용 방식:
  - 기존 게임 TOP 3 딥링크 재패키징(“지금 바로 플레이” 카드형 포스트)
  - 기존 리더보드/유저 기록 캡처 이미지 재활용
- 집행 포맷:
  - 고정 메시지 1개 업데이트
  - 24시간 내 포스트 2회 (런치형 1, 경쟁형 1)

### 채널 B — TikTok/YouTube Shorts (Rented)
- 자산 활용 방식:
  - 기존 게임플레이 영상 클립 3개(15~25초) 재편집
  - 기존 썸네일/텍스트 오버레이 재사용
- 집행 포맷:
  - 48시간 내 Shorts 3개 업로드
  - 캡션 CTA를 Telegram 딥링크로 통일

---

## 2) 실험 가설 3개

### 가설 1 — CTA 문구 실험
- 가설: 기능형 CTA(“지금 플레이”)보다 경쟁형 CTA(“친구 점수 이겨보기”)가 클릭률(CTR)을 높인다.
- A안: “지금 플레이”
- B안: “친구 점수 이겨보기”
- 성공 기준: B안 CTR이 A안 대비 +20% 이상

### 가설 2 — 콘텐츠 훅 실험
- 가설: Shorts 첫 2초에 “기록 도전” 메시지를 넣으면 완시율이 상승한다.
- A안: 일반 인트로(게임 로고)
- B안: 도전형 훅(“10초 안에 이 점수 가능?”)
- 성공 기준: B안 3초 유지율 +15%, 완시율 +10%

### 가설 3 — 유입 전환 동선 실험
- 가설: Telegram 포스트의 딥링크를 단일 CTA로 집중하면 분산 링크 대비 플레이 시작 전환율이 높다.
- A안: 다중 링크(게임 3개 나열)
- B안: 단일 링크(이번 주 추천 1개)
- 성공 기준: B안 플레이 시작률 +15%

---

## 3) 측정 KPI

### 1차 KPI (48시간 핵심)
1. Link CTR (채널별)
2. Game Start Rate (딥링크 클릭 대비 실제 시작)
3. CAC 대체지표: 콘텐츠 1개당 신규 플레이어 수

### 2차 KPI (품질 확인)
4. D1 리텐션(실험 유입 코호트)
5. 공유율(플레이어 1인당 공유 버튼 클릭)
6. 채널 전환율(Rented → Owned: Shorts 유입자의 텔레그램 구독 전환)

### KPI 목표치(48시간)
- Telegram CTR: 4.5% 이상
- Shorts → 딥링크 CTR: 1.8% 이상
- 딥링크 → 게임 시작률: 55% 이상
- 신규 플레이어: 기준선 대비 +25%

## 4) 48시간 실행 체크리스트

###+0h ~ +6h (준비)
- [ ] 기존 게임 3개 우선순위 선정 (최근 리텐션 상위 기준)
- [ ] 기존 영상/이미지 자산 6개 선별
- [ ] UTM 규칙 확정 (`tg_owned`, `shorts_rented`, `cta_a/b`)
- [ ] KPI 추적 시트(시간대/채널/크리에이티브) 생성

###+6h ~ +24h (1차 집행)
- [ ] Telegram 포스트 #1 업로드 (CTA A)
- [ ] Shorts #1, #2 업로드 (훅 A/B 분리)
- [ ] 고정 메시지 업데이트(단일 CTA 링크)
- [ ] 6시간 후 중간 지표 체크(CTR/시작률)

###+24h ~ +36h (2차 집행 + 보정)
- [ ] 성과 낮은 크리에이티브 교체(하위 30% 컷)
- [ ] Telegram 포스트 #2 업로드 (CTA B)
- [ ] Shorts #3 업로드 (상위 훅 복제)
- [ ] 댓글/반응에서 반복 질문 3개 수집 후 카피 반영

###+36h ~ +48h (마감 분석)
- [ ] 채널별 KPI 집계 (CTR, 시작률, 신규 유입)
- [ ] 가설 3개 승패 판정 (Win/Lose/Iterate)
- [ ] 다음 7일 확장안 1개 확정 (승리 조합만 확장)
- [ ] 결과 요약 5줄 작성 및 저장

## 5) 실행 원칙
- 신규 개발 금지: 기능 추가/툴 제작 없이 기존 자산만 재배치
- 의사결정 기준: 감이 아니라 KPI 달성 여부
- 확장 규칙: 48시간 승리 조합만 예산/빈도 확장
