# Humanizer

AI가 쓴 한국어 텍스트에서 "기계 냄새"를 제거한다.

## 1. AI 텍스트 감지 패턴

아래 신호가 3개 이상이면 humanizing 필수:

| 신호 | 예시 |
|------|------|
| **과도한 구조화** | 모든 단락이 주제문→설명→결론 |
| **전환어 반복** | "또한", "그러나", "한편", "이처럼" 매 단락 시작 |
| **균일한 문장 길이** | 15~25자 문장이 반복, 짧은/긴 문장 부재 |
| **감정 없는 서술** | 사실 나열만, 주관·감탄·의문 없음 |
| **클리셰 표현** | "~의 중요성을 간과할 수 없습니다", "~에 있어서" |
| **완벽한 문법** | 문법 오류 0%, 도치·생략·파격 구문 0% |
| **나열 병렬** | "첫째, 둘째, 셋째" 또는 "A뿐만 아니라 B도" 반복 |

## 2. 한국어 특화 문제점

### 번역투 (가장 큰 적)
| ❌ 번역투 | ✅ 자연스러운 한국어 |
|-----------|---------------------|
| ~하는 것이 중요합니다 | ~해야 해요 / ~가 핵심이에요 |
| ~에 대해서 이야기해 보겠습니다 | ~얘기 좀 할게요 |
| ~라는 사실을 알 수 있습니다 | ~인 거죠 / ~더라고요 |
| 이것은 ~입니다 | 이건 ~예요 / ~거든요 |
| ~할 수 있는 기회를 제공합니다 | ~할 수 있어요 |
| ~에 있어서 매우 중요한 역할을 합니다 | ~에서 진짜 중요해요 |
| ~하는 것은 부인할 수 없는 사실입니다 | 맞아요, ~ |

### 과도한 경어/존칭
- "~하시겠습니까" → 블로그에선 "~할까요?" 또는 "~해볼게요"
- "독자 여러분께서는" → "여러분은" 또는 생략
- 문장마다 "~습니다" → "~예요/~거든요/~죠" 섞기

### 부자연스러운 어미 패턴
- **모든 문장 ~합니다 종결** → 해요체/반말/명사형 종결 섞기
- **~것입니다** 남발 → "~거예요", "~할 거고", "~하려고요"
- **의문문 부재** → 중간에 "그런데 왜 그럴까요?" 같은 자문 삽입

## 3. 소설 Humanizing

### 캐릭터 말투 차별화
```
❌ 모든 캐릭터가 동일한 경어체
"저는 그렇게 생각합니다." / "저도 동의합니다."

✅ 캐릭터별 개성
노인: "그게 말이야, 옛날엔 그런 게 없었어."
10대: "ㄹㅇ? 대박인데 그거."
직장인: "아, 그건 좀… 위에서 뭐라 하지 않을까요?"
```

### 구체적 변환 규칙

| 규칙 | 적용 |
|------|------|
| **불완전한 문장** | "그래서 나는—" (말 끊김), "뭐, 그런 거지." |
| **반복/말버릇** | 캐릭터마다 습관적 단어 부여 ("근데", "아니 근데", "솔직히") |
| **비문법적 대사** | "나 어제 걔 봤는데 진짜 웃기더라 아 진짜" |
| **감각 묘사** | 감정 직접 서술 ❌ → 신체 반응으로 ("손이 떨렸다", "목이 막혔다") |
| **문장 길이 변화** | 긴장 장면 = 짧은 문장. 여유 장면 = 길고 느린 문장. |
| **의성어/의태어** | "쾅", "후다닥", "찔끔" — 한국어의 강점 적극 활용 |
| **구어체 삽입** | 내면 독백에 "아, 씨." "뭐야 이게." 같은 날것 표현 |

### 감정 표현 다양화
```
❌ "그녀는 슬펐다."
❌ "그의 눈에 눈물이 고였다." (클리셰)

✅ "입술을 깨물었다. 안 울 거다. 울면 지는 거다."
✅ "코끝이 시큰했다. 아, 됐다. 다 됐다고."
```

## 4. 블로그 Humanizing

### 개인적 어투 강제 삽입
- 첫 문단에 개인 경험 1개 ("저도 처음엔 몰랐는데요")
- 중간에 실패담 또는 의문 ("근데 이게 진짜 맞나? 싶었어요")
- 완벽한 정보 전달 ❌ → 시행착오 과정 노출

### 구체적 변환 규칙

| 규칙 | 적용 |
|------|------|
| **리듬 변화** | 긴 문장 → 짧은 문장 → 한 단어. 반복하지 말 것. |
| **괄호 활용** | "(솔직히 이건 좀 귀찮긴 해요)" |
| **직접 말 걸기** | "혹시 이런 경험 있으세요?" |
| **비격식 전환** | "~입니다" 3문장 연속 금지. 중간에 "~거든요", "~더라고요" |
| **숫자/구체성** | "많이" ❌ → "3번", "2주", "체감상 70%" |
| **의견 표명** | "제 생각엔", "솔직히 말하면", "개인적으로는" |
| **불확실성 인정** | "이건 확실하진 않은데", "틀릴 수도 있지만" |

### Before/After: 블로그

```
❌ BEFORE (AI 냄새)
Docker는 컨테이너 기반의 가상화 기술입니다. 이를 통해 개발자는
애플리케이션을 일관된 환경에서 실행할 수 있습니다. 또한 배포
과정을 간소화할 수 있다는 장점이 있습니다. 따라서 많은 기업에서
Docker를 채택하고 있습니다.

✅ AFTER (사람 냄새)
Docker 처음 접했을 때 솔직히 뭔 소린가 했어요. 컨테이너?
가상화? 그냥 서버에 올리면 되는 거 아닌가?

근데 한번 써보니까 아. 이래서 다들 쓰는구나. "내 컴에서는
되는데요?" 이 말을 더 이상 안 하게 되더라고요.

환경 세팅에 하루 날려본 적 있으시죠? (저는 3번쯤 날렸습니다)
그게 docker-compose up 한 줄이면 끝나요. 진짜로.
```

### Before/After: 소설

```
❌ BEFORE (AI 냄새)
민수는 카페에 들어갔다. 그는 커피를 주문했다. 잠시 후 그는
창밖을 바라보며 생각에 잠겼다. 그의 마음은 복잡했다. 어제
있었던 일이 계속 떠올랐기 때문이다.

✅ AFTER (사람 냄새)
카페 문을 밀었다. 에어컨 바람이 확 끼얹어졌다.

"아메리카노 하나요. 얼음 많이."

자리 잡고 앉았는데, 아까 그 표정이 또. 씨발. 자꾸 떠오른다.
민수는 빨대를 씹었다. 벌써 세 번째 씹는 거다.

창밖에 비가 올 것 같았다. 아니, 이미 오고 있나.
```

## 5. 제출 전 체크리스트

텍스트 완성 후 아래 항목 전수 검사:

- [ ] "또한/그러나/따라서"가 3회 이상 등장하지 않는가?
- [ ] 모든 문장이 "~습니다/~입니다"로 끝나지 않는가?
- [ ] 번역투 표현이 없는가? ("~에 있어서", "~하는 것이 중요")
- [ ] 문장 길이에 변화가 있는가? (짧은 문장 최소 20%)
- [ ] 개인적 의견/경험/감정이 1개 이상 포함되어 있는가?
- [ ] 소리 내어 읽었을 때 자연스러운가?
- [ ] (소설) 대사가 캐릭터별로 구분 가능한가?
- [ ] (소설) "그는/그녀는" 시작 문장이 연속 2회 이상 없는가?
- [ ] (블로그) 독자에게 직접 말 거는 문장이 1개 이상 있는가?
- [ ] 의문문이 최소 1개 있는가?

## 6. 금지어/금지 패턴

사용 시 즉시 교체:

| 금지 | 대체 |
|------|------|
| ~의 중요성 | 왜 ~가 중요하냐면 |
| ~를 살펴보겠습니다 | ~를 볼게요 / ~얘기할게요 |
| ~라고 할 수 있습니다 | ~인 거죠 / ~예요 |
| 다양한 ~가 있습니다 | 몇 가지 있는데요 |
| ~에 대해 알아보겠습니다 | ~ 한번 파봅시다 |
| 결론적으로 | (삭제하고 자연스럽게 마무리) |
| 요약하자면 | (삭제하고 자연스럽게 마무리) |
| ~하는 것은 필수적입니다 | ~해야 돼요 / 이거 안 하면 큰일나요 |
