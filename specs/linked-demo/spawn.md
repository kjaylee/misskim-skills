# linked-demo 스폰 지시문

## 현재 상태
초기 생성

## 작업 목표
기능 구현 작업 수행

## 범위
작업 범위 안에서 구현, 테스트, 검증까지 완료

## 테스트 또는 검증 기준
관련 테스트 작성 또는 갱신, 실행 결과 기록

## 완료 조건
코드 변경과 검증 증거가 함께 남는다

## 산출물 경로
specs/linked-demo/plan.md, specs/linked-demo/spawn.md, specs/linked-demo/spawn-ready.md, .state/pipelines/linked-demo.json

## 금지 사항
테스트 없는 완료 보고 금지, 범위 밖 리팩터 금지

## 실행 원칙
- 묻지 말고 범위 안에서 먼저 실행
- 범위 밖 변경 금지
- 테스트와 검증 증거 없이 완료 보고 금지

## 레드팀
- 공격 하나: 범위가 커져서 원래 작업이 흔들릴 수 있음
- 공격 둘: 검증이 약하면 빈 문서만 늘어날 수 있음
- 방어: 범위, 테스트, 완료 조건, 산출물 경로를 강제해서 빈 작업을 줄인다
- 합의: 위험 수용

## 최종 보고 형식
- 완료한 일
- 검증 결과
- 남은 위험
